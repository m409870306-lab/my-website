# 婚纱礼服 AI 试穿小程序设计方案

日期：2026-06-21

## 目标

设计一款界面整洁、高级、可信的婚纱礼服 AI 试穿小程序。顾客上传本人照片后，从商家上传的婚纱礼服图库中选择款式，系统生成最终试穿效果图。

核心优先级：

1. 保证最终图与顾客本人相似，保留脸部、发型、肤色、气质和自然体态。
2. 在相似度基础上，根据不同体型调整婚纱上身尺寸和视觉效果。
3. 只做轻度美化，避免生成图和本人差距过大。
4. 成片要适合婚纱馆咨询、预约和到店试穿转化。

## 已确认方向

视觉方向：柔和高级婚纱店风格。主色使用象牙白、香槟金、暖灰和深墨黑，整体像高端婚纱馆工具，不做娱乐滤镜感。

接口方向：优先使用 OpenAI 图像编辑能力做 MVP。官方文档说明 OpenAI API 可用 GPT Image 模型生成和编辑图片，当前图像模型包含 `gpt-image-2`；图像编辑接口支持传入多张参考图并使用 `images.edit` 生成结果。相关参考：

- OpenAI Image generation guide: https://developers.openai.com/api/docs/guides/image-generation
- OpenAI Images API reference: https://developers.openai.com/api/reference/resources/images

## 用户流程

1. 首页
   - 展示“上传本人照，选择礼服，生成试穿效果”的主行动。
   - 语气高级、直接，不夸大效果。

2. 上传本人照
   - 支持最多 3 张照片。
   - 第 1 张为必传正面照，用于主体姿态、脸部和发型参考。
   - 第 2 张为可选五官参考，用于增强脸型、发型和五官稳定性。
   - 第 3 张为可选全身比例照，用于辅助肩颈、身材比例和站姿。
   - 页面提示照片要求：光线清晰、无遮挡、正面为主、不要使用强滤镜。

3. 可选体型信息
   - 字段：身高、体重、平时尺码、期望效果。
   - 平时尺码可选 S / M / L / XL。
   - 期望效果默认“自然轻美化”。
   - 信息可跳过，但界面提示“信息越完整，礼服比例越稳定”。

4. 选择礼服
   - 礼服来自商家后台上传的婚纱礼服照片。
   - 每件礼服保留编号、名称、类型、版型、材质和适合场景。
   - MVP 中先支持单件礼服选择。

5. 生成试穿图
   - 生成中状态展示正在处理：身份相似度、体型比例、礼服贴合、自然轻美化。
   - 不在结果图上叠加解释性提示。

6. 结果页
   - 试穿效果图整版展示。
   - 标题和礼服编号放在图片上方独立区域，不遮挡人物头部或主体。
   - 图片下方保留“保存并咨询”“重新生成”“预约试穿”等操作。
   - 结果页不显示“相似度优先 / 按体型调整 已应用”这类提示卡。

## 体型适配要求

生成结果不能只是把婚纱贴到人物上，而要根据顾客体型改变礼服上身效果。

必须体现的变化：

- 肩宽和袖口位置跟随顾客身体比例变化。
- 胸口贴合度根据上身比例变化。
- 腰线位置根据身高感、腰身和礼服版型变化。
- 裙摆宽度、裙身体积和拖尾比例根据体型与站姿变化。
- 不默认统一瘦身，只允许自然轻美化。

界面和文案中要避免承诺真实尺码精确推荐。结果应定位为“试穿视觉预览”，真实尺码仍以到店试穿和顾问建议为准。

## AI 生成流程

MVP 生成流程：

1. 前端收集输入
   - 顾客照片 1-3 张。
   - 选中的礼服照片。
   - 礼服结构标签。
   - 可选体型信息。

2. 后端构造图像编辑请求
   - 使用顾客照片作为身份和体型参考。
   - 使用礼服照片作为服装参考。
   - 提示词强调身份保留、体型适配、礼服材质、轻度美化和禁止遮挡。

3. 调用 OpenAI 图像编辑接口
   - 推荐 MVP 使用 `gpt-image-2` 的图片编辑能力。
   - 图片输入较多时成本会升高，需记录每次生成的输入数量、图片尺寸和结果状态。

4. 结果处理
   - 保存生成图、礼服编号、输入摘要、生成时间和状态。
   - 失败时返回明确原因，如照片不清晰、人物遮挡、礼服图片不合适或接口失败。

## 提示词原则

生成提示词必须包含以下约束：

- 保留顾客脸部相似度、发型、肤色和整体气质。
- 使用礼服参考图的版型、领口、袖型、材质、裙摆和拖尾。
- 根据顾客体型调整婚纱比例，不使用统一模板。
- 保持自然站姿和真实婚纱摄影质感。
- 只做轻度美化，不改变身份特征。
- 不遮挡头部和脸部。
- 不生成文字、水印、品牌标志或夸张背景。

## 页面结构

MVP 页面：

- 首页
- 上传本人照页
- 体型信息页
- 礼服选择页
- 生成中页
- 试穿结果页

商家侧后续页面：

- 礼服上传
- 礼服列表
- 礼服分类和标签
- 顾客试穿记录

商家侧可以放到第二阶段，不阻塞 MVP 的顾客端验证。

## 数据结构

顾客试穿任务：

- `id`
- `customerImages`: 1-3 张图片地址
- `bodyInfo`: 身高、体重、尺码、期望效果
- `dressId`
- `dressImage`
- `generationStatus`: pending / processing / succeeded / failed
- `resultImage`
- `failureReason`
- `createdAt`

礼服：

- `id`
- `name`
- `code`
- `image`
- `category`
- `silhouette`
- `fabric`
- `neckline`
- `sleeve`
- `train`
- `isActive`

## 错误处理

上传阶段：

- 图片过小、模糊、非人像、多人同框、脸部遮挡时提示重传。
- 礼服图过暗、被模特姿势严重遮挡、只有局部细节时提示商家换图。

生成阶段：

- 接口超时：允许重试。
- 相似度明显不足：提供“重新生成”。
- 礼服变形明显：建议更换礼服图或补充体型信息。
- 内容安全或图片不合规：提示无法生成，不展示技术错误。

## 验收标准

功能验收：

- 顾客可上传 1-3 张本人照片。
- 顾客可跳过或填写体型信息。
- 顾客可选择一件礼服生成试穿图。
- 结果页整版展示试穿图，文字不遮挡头部和人物主体。
- 生成失败时有清晰提示和重试入口。

效果验收：

- 生成图优先保留本人相似度。
- 不同体型顾客穿同一件婚纱时，礼服视觉尺寸和比例有合理差异。
- 轻度美化不改变顾客身份特征。
- 礼服材质、版型和拖尾在结果图中可辨认。

体验验收：

- 界面干净、高级、适合婚纱馆场景。
- 上传和生成流程不显得复杂。
- 结果页能直接支持保存、咨询和预约。
